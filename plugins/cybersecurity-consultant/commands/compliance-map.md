Map an organization's security controls against one or more compliance frameworks.

Gather:
- Target framework(s): SOC 2, ISO 27001, CMMC, NIST 800-53, HIPAA, PCI-DSS, GDPR
- Organization's current control inventory (or help them build one)
- Existing policies and procedures
- Technical controls in place
- Areas of known non-compliance

Produce a compliance mapping:

## 1. Framework Overview
- Description of the chosen framework
- Applicability to the organization
- Certification/attestation process overview
- Timeline and effort estimate for compliance

## 2. Control Mapping Matrix
| Framework Requirement | Control ID | Description | Current Status | Gap | Evidence Needed |
|----------------------|-----------|-------------|---------------|-----|----------------|
| [Requirement] | [ID] | [What's needed] | Met/Partial/Not Met | [Gap description] | [Required evidence] |

## 3. Gap Analysis Summary
- Controls fully met: [count and %]
- Controls partially met: [count and %]
- Controls not met: [count and %]
- Overall readiness score

## 4. Cross-Framework Mapping
If multiple frameworks, show where controls overlap:
| Control | SOC 2 | ISO 27001 | NIST 800-53 |
|---------|-------|-----------|-------------|
| Access control policy | CC6.1 | A.9.1.1 | AC-1 |

## 5. Remediation Plan
For each gap:
- Control requirement
- Current state
- Required actions
- Responsible party
- Effort estimate
- Priority

## 6. Evidence Collection Guide
For each control area, specify:
- What evidence auditors expect
- How to collect and organize it
- Retention requirements
- Automation opportunities

Save as a comprehensive compliance mapping document.
