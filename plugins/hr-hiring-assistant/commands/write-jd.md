# /write-jd

Write a compelling, inclusive job description for any role. See the **job-description-writer** skill for the full framework, role-specific templates, and inclusive language guidelines.

## Usage

```
/write-jd $ARGUMENTS
```

## What I Need From You

- **Role title**: What's the position?
- **Department/team**: Where does this role sit?
- **Level**: Junior, Mid, Senior, Lead, Manager, Director, VP?
- **Location**: In-office, hybrid, remote? If remote, any timezone/location restrictions?
- **Salary range**: What's the budget? (I'll recommend including it — postings with salary get 30%+ more applicants)
- **Key responsibilities**: What will this person actually do day-to-day?

Don't have all the details? I'll ask smart questions to fill in the gaps.

## Output

A complete, ready-to-post job description with:
- Compelling hook that sells the role (not corporate boilerplate)
- Specific responsibilities (what a typical week looks like)
- Requirements split into Required vs. Nice-to-Have (improves diversity of applicants)
- Salary range and benefits
- Company context and team info
- Inclusive language throughout
