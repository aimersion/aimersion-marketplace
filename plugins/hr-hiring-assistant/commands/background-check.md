# /background-check

Set up a compliant background check process or manage specific check results. See the **background-checks** skill for the full workflow, adverse action process, and ban-the-box compliance.

## Usage

```
/background-check $ARGUMENTS
```

## Modes

**Setup:** "What background checks should we run for a [role type]?"
**Process:** "A background check came back with findings — what do I do?"
**Reference checks:** "Help me run reference checks for [candidate]"

## Output

Depending on the mode:
- **Setup:** Recommended check types for the role, provider options, consent form checklist
- **Process:** Step-by-step adverse action process (FCRA-compliant), individualized assessment framework
- **Reference checks:** Question list, red flag indicators, documentation template
