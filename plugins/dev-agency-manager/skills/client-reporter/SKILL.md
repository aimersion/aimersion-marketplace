# Client Reporter

Create polished, professional client-facing project status communications for development agency engagements.

## Reporting Principles

1. **Lead with outcomes, not activities.** Clients care about progress toward their goals, not your internal process.
2. **Be transparent about risks.** Early flagging builds trust; surprises destroy it.
3. **Quantify progress.** Use metrics, percentages, and concrete numbers.
4. **Keep it scannable.** Executives read summaries. Include detail for those who want it.
5. **End with clear asks.** If you need decisions or input, make it obvious.

## Report Types

### Sprint Report (every 2 weeks)
- Sprint goal and achievement status
- Completed features/stories
- Sprint metrics (velocity, planned vs. actual)
- Demo highlights
- Next sprint plan
- Blockers requiring client action

### Monthly Executive Summary
- Project health dashboard (RAG status)
- Budget utilization (hours or dollars)
- Key milestones achieved and upcoming
- Risk register update
- Strategic recommendations

### Milestone Report
- Milestone definition and acceptance criteria
- Deliverables completed
- Quality metrics (test coverage, defect density)
- Acceptance testing results
- Next milestone preview

For detailed report templates and formatting, read `references/reporting-templates.md`.

## Status Indicators

Use consistent RAG (Red-Amber-Green) status:
- **Green**: On track, no significant issues
- **Amber**: At risk, issues identified but mitigatable
- **Red**: Off track, requires immediate intervention or scope/timeline change

Apply RAG to: Overall project, Schedule, Budget, Quality, Team

## Output Guidelines

When creating client reports:
- Use the client's preferred format if specified
- Include data visualizations when data is available
- Always include a clear executive summary
- Separate "FYI" information from "action required"
- Proofread for professionalism — this represents the agency
- Save as a formatted, downloadable document
