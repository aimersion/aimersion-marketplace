Conduct a thorough audit of the user's SaaS onboarding experience.

Gather context:
- Product type and target user persona
- Current onboarding steps (or access to the product)
- Time-to-value benchmarks they're tracking
- Current activation rate and definition of "activated user"
- Known drop-off points

Deliver an onboarding audit report:

## 1. Onboarding Flow Map
Document every step from signup to activation. For each step, note:
- Action required from user
- Estimated time to complete
- Friction level (Low/Medium/High)
- Drop-off risk

## 2. Time-to-Value Analysis
Calculate the current time-to-first-value. Compare against SaaS benchmarks:
- PLG products: < 5 minutes ideal
- Mid-market B2B: < 1 day ideal
- Enterprise: < 1 week ideal

## 3. Activation Funnel
Build a step-by-step activation funnel showing conversion rates between steps. Identify the biggest drop-off points.

## 4. Friction Audit
For each friction point identified:
- Describe the friction
- Classify: necessary vs. removable
- Recommend specific fix
- Estimate impact on activation rate

## 5. Product Tour Recommendations
Design an optimized onboarding sequence:
- Welcome experience
- Core feature introduction order
- Progressive disclosure strategy
- Milestone celebrations
- Re-engagement triggers for stalled users

## 6. Measurement Framework
Define the metrics to track onboarding success:
- Activation rate by cohort
- Time-to-value
- Feature adoption sequence
- Onboarding completion rate

Save the audit as a downloadable document.
