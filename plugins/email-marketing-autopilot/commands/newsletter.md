Draft a complete email newsletter about: "$ARGUMENTS"

Follow the newsletter-writer skill workflow:

1. If the user provided a topic, build the newsletter around it. If no topic given, ask what the newsletter should cover.

2. Select the best newsletter format for this content:
   - **Curated Digest** — for link roundups, industry news (5-8 short items)
   - **Editorial** — for thought leadership, deep analysis (one main article)
   - **Hybrid** (default) — hero story + 2-3 supporting sections
   - **Product Update** — for feature announcements, changelogs

3. Generate 3 subject line options using different formulas:
   - Curiosity-driven (opens a loop)
   - Benefit-driven (promises specific value)
   - Direct/Clear (states what's inside)
   Include character counts for each.

4. Write complementary preview text (50-90 chars) that adds info beyond the subject.

5. Compose the full newsletter:
   - **Hero section**: Lead story with 2-3 short paragraphs + primary CTA
   - **2-3 content sections**: Mix section types (curated links, tips, spotlight, quick hits)
   - **Order by engagement potential**: Highest-click section right after hero
   - **Footer**: Personal sign-off (human name), reply prompt, preference center link, unsubscribe

6. Follow these non-negotiable standards:
   - Total: 400-700 words (hybrid format)
   - Paragraphs: 2-3 sentences MAX
   - Every section has at least one link/CTA
   - Minimum 3, maximum 8-10 links total
   - Tone: Smart, approachable, like a colleague sharing what matters
   - [First Name] personalization where natural
   - Mobile-first: assume 60%+ open on phones

7. End with:
   - Best send day/time (Tue-Thu, 9-11am for B2B; Tue/Thu for B2C)
   - Segmentation suggestions (should any segments get a variant?)
   - A/B test idea for this specific send
   - Key metric to watch for this newsletter

Format with clear section headers so it's ready to paste into an email platform.
