Create compliance documentation for: $ARGUMENTS

Follow the compliance-documenter skill methodology:

1. Determine documentation type: client meeting notes, suitability analysis, or ADV update.
2. For meeting notes:
   - Gather details: date, attendees, meeting type, topics discussed
   - Document client situation updates, recommendations made with rationale, client decisions, and action items
   - Use objective, contemporaneous language suitable for regulatory review
3. For suitability documentation:
   - Capture full client profile (age, income, net worth, risk tolerance, objectives, time horizon)
   - Document the specific recommendation with cost/fee disclosure
   - Provide explicit suitability rationale connecting recommendation to client profile
   - Note alternatives considered and risks disclosed
   - Include concentration check
4. For ADV updates:
   - Walk through the annual update checklist
   - Identify material changes requiring client notification
   - Document delivery dates

Format the output using the appropriate compliance template. Flag any areas where additional information is needed from the advisor.
