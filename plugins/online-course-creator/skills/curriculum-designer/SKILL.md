# Curriculum Designer

Design complete online course curricula from topic selection through assessment strategy. Build courses that are pedagogically sound, engaging, and optimized for online delivery.

## Course Architecture

### Course Planning Framework
1. **Audience Analysis** — Who is the learner? What do they already know? What do they need?
2. **Transformation Statement** — "By the end of this course, students will go from [current state] to [desired state]"
3. **Learning Outcomes** — 3-7 measurable course-level outcomes
4. **Module Mapping** — Break the transformation into sequential modules
5. **Assessment Strategy** — How students demonstrate mastery at each stage

### Learning Objectives (Bloom's Taxonomy)
Write objectives using measurable action verbs:
- **Remember**: Define, list, recall, identify, name
- **Understand**: Explain, describe, summarize, interpret, compare
- **Apply**: Use, implement, demonstrate, solve, calculate
- **Analyze**: Differentiate, organize, compare, deconstruct, examine
- **Evaluate**: Judge, critique, assess, justify, recommend
- **Create**: Design, construct, produce, develop, formulate

Format: "By the end of this module, learners will be able to [verb] [what] [context/conditions]."

### Module Structure (Recommended)
Each module follows this pattern:
1. **Hook** (5 min) — Story, question, or problem that creates curiosity
2. **Core Content** (15-30 min) — Instruction broken into 5-10 minute segments
3. **Application** (10-20 min) — Practice exercise, case study, or project step
4. **Assessment** (5-10 min) — Quiz, reflection, or peer activity
5. **Bridge** (2 min) — Preview of next module, creating anticipation

### Course Length Guidelines
- **Mini-course** (1-2 hours): 3-5 modules, single focused skill
- **Standard course** (4-8 hours): 6-12 modules, comprehensive topic
- **Flagship course** (12-30+ hours): 15-30 modules, deep mastery program
- **Cohort program** (4-12 weeks): Weekly modules with live components

## Module Sequencing Strategies

### Linear Progression
Best for: skill-building courses where each concept builds on the prior one
- Module 1: Foundation → Module 2: Basic Application → Module 3: Advanced Techniques

### Hub and Spoke
Best for: courses where modules are somewhat independent
- Core module teaches the framework; subsequent modules apply it to different contexts

### Problem-First (Case-Based)
Best for: professional development and business courses
- Present a real-world problem → teach concepts needed to solve it → apply solution

### Project-Based
Best for: creative and technical courses
- Each module contributes a piece to a capstone project

## Assessment Design

### Formative Assessments (During Learning)
- Knowledge checks after each video segment (3-5 questions)
- Reflection prompts ("How would you apply this in your situation?")
- Mini-exercises with immediate feedback
- Discussion posts comparing approaches

### Summative Assessments (End of Module/Course)
- Module quizzes (10-15 questions, mix of types)
- Project milestones with rubrics
- Peer review assignments
- Capstone project or portfolio

### Question Types
- **Multiple choice**: Test recognition and basic understanding
- **Short answer**: Test recall and explanation
- **Scenario-based**: Test application to real situations
- **Project-based**: Test creation and synthesis
- **Peer review**: Build community and develop evaluation skills

## Content Mapping Template

For each module, define:
- Module title and number
- Learning objectives (2-3 per module)
- Content outline (topics and subtopics)
- Content format (video, text, audio, interactive)
- Estimated completion time
- Required resources or materials
- Assessment method
- Prerequisites (which modules must come first)
