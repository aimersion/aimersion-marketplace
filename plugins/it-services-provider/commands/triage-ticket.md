Triage an incoming IT support ticket with professional MSP-grade classification.

Gather from the user:
- Ticket description or user-reported issue
- Affected user(s) and their role/department
- Client name and SLA tier
- Systems or applications affected
- Time of occurrence and duration
- Any error messages or screenshots

Produce a triage assessment:

## 1. Classification
- **Category**: Hardware / Software / Network / Security / Access / Email / Printing / Other
- **Subcategory**: Specific issue type
- **Priority**: P1 (Critical) / P2 (High) / P3 (Medium) / P4 (Low)
- **SLA Response Time**: Based on client tier and priority
- **SLA Resolution Target**: Based on client tier and priority

## 2. Impact Assessment
- Users affected (single user, department, entire organization)
- Business impact (revenue-affecting, productivity loss, security risk)
- Is there a workaround available?

## 3. Initial Diagnosis
- Likely root cause based on symptoms
- Troubleshooting steps to try first
- Escalation path if initial steps fail
- Similar known issues from knowledge base

## 4. Assignment Recommendation
- Recommended technician tier (L1 / L2 / L3 / vendor escalation)
- Required skills or certifications
- Estimated resolution time
- Parts or licensing that may be needed

## 5. Client Communication
Draft a professional first-response message to the end user confirming receipt, priority, and expected response time.
