Draft a professional, empathetic customer response using the response-drafter skill.

**Ticket / Context**:
$ARGUMENTS

Follow this process:

1. Analyze the ticket to understand the issue, severity, customer tone, and account context.
2. Determine the appropriate channel (email, chat, in-app) and tone (urgent, empathetic, friendly, professional).
3. Draft the response following the acknowledgment → context → resolution → closing framework.
4. Avoid all anti-patterns listed in the response-drafter skill (generic empathy, blame language, empty promises).
5. Include internal notes with any follow-up actions, caveats, or context the support agent should know.

If the ticket suggests the customer is frustrated or angry, use de-escalation techniques — extra empathy, concrete actions, and personal accountability.

Present the response in the structured output format from the response-drafter skill.
